# GitHub Project Board Setup

## Ziel
Das Board ist das operative Steuerungszentrum für den Dinova-Rollout.

## Empfohlene Spalten / Statuswerte
- Backlog
- Diese Woche
- In Arbeit
- Waiting for Client
- Review
- Done

## Empfohlene Custom Fields
### 1. Priorität
- P1
- P2
- P3

### 2. Phase
- Phase 0 – Foundation
- Phase 1 – Shopify Core Setup
- Phase 2 – Sortimentsaufbau & Content
- Phase 3 – Shopify Go-Live
- Phase 4 – Marktplatzvorbereitung
- Phase 5 – Marktplatz Go-Live & Skalierung

### 3. Owner
Freitext oder Personenzuweisung.

### 4. Deadline
Datumsfeld.

### 5. Risiko
- niedrig
- mittel
- hoch

## Steuerungslogik
- Alles, was nicht terminiert und zugewiesen ist, bleibt Backlog.
- Alles mit externer Abhängigkeit geht in `Waiting for Client`.
- Pro Woche maximal 5–8 echte Arbeitspakete parallel.
- Entscheidungen und Risiken bekommen eigene Issues.
